Archive consultation: technical aspects

The Archive is designed to be consulted using common hardware without special software -- certainly without installing any software written by us! This page discusses some of the issues that may arise.

The client machine

We have successfully tested access to the archive on clients running the following OS-browser combinations:

  • Windows (95/98/NT/2000/XP) with the following browsers:
    • Netscape (versions 6 and higher)
    • Mozilla Firefox
    • Internet Explorer (versions 4 and higher)
  • Macintosh (MacOS8, MacOS9, MacOSX) using:
    • Netscape (versions 6 and higher)
    • Mozilla Firefox
  • Linux using:
    • Netscape 7
    • Mozilla Firefox

For basic access to the sound resources, the default browsers and media players on recent machines are usually satisfactory. For full-featured access (see below), we have tested the following OS/browser/media player combinations:

  • Windows2OOO/XP/Firefox/Java Media Framework
  • Windows2OOO/XP/Explorer6/QuickTime Active-X
  • Windows2OOO/XP/Netscape7.1/QuickTime Plugin
  • MacOSX/Mozilla 1.7/QuickTime Plugin

Text display

Many of the Archive documents are transcribed in the International Phonetic Alphabet, and a few in Cyrillic or Devanagari. The special characters used (which are relatively few in the case of IPA transcriptions) will not be displayed unless a font that implements the relevant parts of the Unicode is installed on the client machine. We have tested the following fonts:

  • Lucida Sans Unicode (bundled with WNT and W98).
  • Arial Unicode MS (bundled with Office 2000 and XP)
  • Code2000 (shareware).
  • Caslon (for Linux).

For up-to-date information on Unicode fonts see: http://www.alanwood.net/unicode/index.html.

Mac users: Unicode fonts are directly viewable only on MacOSX. For Mac OS9 and earlier, after choosing a text, choose the "pdf" display option in the "choose what you want to see" window, for basic browsing. No font installation is required.

Sound

Sound resources are available in both uncompressed (wav) and compressed MPEG I Layer 3 (mp3) formats. Choose mp3 instead of wav (the default) to shorten transfer times. The client browser must have access to an utility capable of reading this format. The choice is offered when you have chosen a text for browsing.

The Archive interface provides for three different means of access to the sound:

  • using a plug-in or independent application able to decode mp3 and/or wav sound:
    This method has the advantage of simplicity: it works on all platforms, and many recent systems will already have the necessary tools installed. For technical reasons we are obliged to limit access by this method to a single segment of sound (usually corresponding to a sentence in our markup) at a time. Recommended for casual browsing.
  • using one of the tools (plugin or activeX) "Windows Media Player" or "QuickTime player".
    The interface gives access to the sound either segment by segment or continuously, with the corresponding transcription highlighted as the sound is played.
    Free versions of WMP are available here.
    Free versions of QuickTime (for Windows or Mac) are available here.
  • using the Java Media Framework (JMF) API:
    This is our favorite multi-platform solution. It also gives access to the sound either segment by segment or continuously.
    To use this method, you must first have installed a Java Virtual Machine from Sun (we use the Java runtime executive j2se jre v. 1.4.2). Then install one of the four implementations of the JMF API (for Windows, Linux, Solaris, and Java) from the Sun JMF website. Don't forget to also get the plug-in needed for JMF to handle MP3.

You may have to configure your browser or your player so that your system actually uses the player you have installed to read the audio formats delivered by the archive (*.mp3 and *.wav files and the audio/mpeg and audio/wav MIME formats). Depending on the setup, the sound will be played in the browser window (plugin) or in an external window (standalone).

Downloading for local access

The annotation files (xml) and the sound files (wav or mp3) can be downloaded from the "metadata search" display or from the metadata displayed when you click on the title of a document in the browsing window. No interface is provided for browsing texts locally.

Practical advice: Loading the sound over the web can be time-consuming. First-time users should try the Hayu text "Brothers", which is very short. The Nemi (New Caledonia) text "Bac and Dangem" is also relatively short and has no IPA characters. Choose the media file format "audio/mpeg" instead of "audio/x-wav" for faster transfer.
