Instructions (Comment consulter l'archive)

Le système a été testé avec succès sur les configurations (OS-browser) suivantes :

  • Windows (95/98/NT/2000/XP) avec :
    • Netscape (versions 6 et suivantes)
    • Mozilla Firefox
    • Internet Explorer (versions 4 et suivantes)
  • Macintosh (MacOS8, MacOS9, MacOSX) avec :
    • Netscape (versions 6 et suivantes)
    • Mozilla Firefox
  • Linux avec Mozilla Firefox et Netscape 7.

Toutes les conbinaisons ne permettant pas toutes les fonctionnalités , nous donnons ci-dessous quelques exemples de configurations que nous utilisons actuellement et qui nous donnent entière satisfaction:

  • Windows2000/Mozilla Firefox/Java Media Framework.
  • Windows2000/IE 6/QuickTime activeX .
  • Windows2000/Netscape 7.1/QuickTime Plugin.
  • MacOS X/Mozilla 1.7/QuickTime Plugin.

Le rendu du texte

Certains documents ont été transcrits en Alphabet Phonétique International (API), d'autres en cyrillique ou en devanagari. Sans l'installation d'une police Unicode implémentant ces alphabets, certains caractères ne s'afficheront pas correctement. Nous avons testé les polices suivantes :

  • Lucida Sans Unicode (livrée avec WNT et W98).
  • Arial Unicode MS (livrée avec Office 2000...).
  • Code2000 (distribuée en shareware).
  • Caslon (sur Linux)

Pour un recensement plus exaustif des polices en fonction des alphabets et des plates-formes consultez le site: http://www.alanwood.net/unicode/index.html

Utilisateurs Mac : Les fontes Unicode fonctionnent en OS X seulement.

 

Le rendu du son

Pour des raisons d'économie de temps de transfert, toutes les données sonores sont compressées au format MPEG I Layer 3 (mp3). Il faut donc qu'un outil permettant de lire ce format soit installé sur la machine client.

Trois méthodes d'accès au son sont prévues :

  • en utilisant une application ou un plugin sachant jouer le son au format mp3.
    Cette méthode a l'avantage de la simplicité : elle fonctionne sur toutes les plate-formes, et beaucoup d'installations récentes auront déjà le logiciel nécessaire installé ; sinon, il est facile d'y remédier. Pour des raisons techniques nous avons limité cette interface à l'écoute d'un segment du son à la fois. C'est la méthode recommandée pour une consultation occasionnelle, pour des plates-formes anciennes ou pour des connexions à faible débit.
  • en utilisant un des deux plugins ou activeX suivants: "Windows Media Player" ou "QuickTime player".
    Cette solution permet à l'utilisateur d'accéder au son soit segment par segment, soit en continu, la transcription correspondante étant surbrillée dynamiquement.
  • en utilisant Java Media Framework (JMF).
    Cette méthode requiert l'utilisation de la machine virtuelle Java de Sun ainsi que l'instalation de l'outil JMF et de son plugin pour lire le format mp3. Elle permet à l'utilisateur d'accéder au son, soit segment par segment, soit en continu, la transcription correspondante étant surbrillée dynamiquement.

 

Configuration nécessaire à la consultation avec des players/plugins mp3

Cliquez ici pour tester si votre système est prêt à lire du mp3. Si oui, vous n'avez pas besoin d'installer un outil.

Sinon, il existe une variété d'outils adéquats pour les plate-formes courantes, par exemple : QuickTime, Real Player, Windows Media Player, Winamp, etc. Il suffit d'installer l'un de ces outils. Il peut éventuellement être nécessaire de signaler à votre browser, via son menu de configuration, qu'il doit appeler l'outil installé pour lire les fichiers avec l'extension mp3 et le type MIME (officieuse) audio/mpeg. Selon l'outil, cette configuration se fera dans les préférences du browser ou dans celles du player. Enfin, selon l'outil et sa configuration, le fichier son sera joué dans une fenêtre externe au browser (standalone application) ou bien dans la fenêtre du browser (plugin).

 

Configuration nécessaire à la consultation avec JMF

Il s'agit de notre outil multi-plate-forme recommandé. Sun propose 4 implementations de JMF (pour Windows, pour Linux, pour Solaris et pour Java) sur leur site ainsi qu'un plugin pour la prise en compte du format mp3.
Vous devez auparavant installer une Machine Virtuelle Java de Sun: par exemple (la version j2se 1.4.2 jre) si vous ne l'avez pas déjà fait.

 

Configuration nécessaire à la consultation avec Windows Media Player

Il faut installer une version récente de Windows Media Player. Vous pouvez télécharger une version gratuite sur le site de Microsoft.

Utilisateurs Linux : Windows Media Player n'existe pas sur Linux.

 

Configuration nécessaire à la consultation avec QuickTime Player

Il faut installer une version récente de QuickTime. Vous pouvez télécharger une version gratuite sur le site de QuickTime

Utilisateurs Linux : QuickTime n'existe pas sur Linux.

 

Téléchargement pour un accès en local

Les fichiers d' enregistrement (wav ou mp3) ainsi que les fichiers d'annotation (xml) de chaque document d'archive peuvent être téléchargés. Les adresses de ces fichiers sont accessibles dans les métadonnées qui sont fournies pour chaque texte. Lorsqu'un texte a été choisi pour consultation, cliquez sur son titre. Aucune interface n'est fournie pour la consultation en local.

Conseil pratique : Pour une première consultation de l'archive, commencez par le texte hayu "Manifesto", qui est très court et donc (relativement) rapide à charger. Le texte némi "Bac et Dangem" (Nouvelle-Calédonie) est relativement bref et sans caractères API.
